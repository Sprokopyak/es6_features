ES6 features

Feature Descrption Code
Let + Const Block-scoped binding constructs. let is the new var. const is single-assignment. Static restrictions prevent use before assignment.
Destructuring Destructuring assignment is a special syntax that allows us to “unpack” arrays or objects into a bunch of variables, as sometimes they are more convenient.
Rest Rest parameters are indicated by three dots … preceding a parameter. Named parameter becomes an array which contain the rest of the parameters.
Spread The spread is closely related to rest parameters, because of … (three dots) notation. It allows to split an array to single arguments which are passed to the function as separate arguments.
Default value ES6 gives us a way to set default function parameters. Any parameters with a default value are considered to be optional.
for…of Using for…of we can iterate over the values in an iterable, like an array, string, arguments, object
Map Map - колекція для зберігання записів виду ключ: значення На відміну від об'єктів, в яких ключами можуть бути тільки стрінга, в Map ключем може бути довільне значення навіть намбер map.delete (keys) видаляє запис з ключем key, повертає true, якщо такий запис був, інакше false.
map.clear () - видаляє всі записи, очищає map.
map.keys () - повертає ітеріруемий об'єкт для ключів,
map.values () - повертає ітеріруемий об'єкт для значень,
Set Set - колекція для зберігання безлічі значень, причому кожне значення може зустрічатися лише один раз, тобто має бути унікальним, має ті ж методи, що і Map
WeakMap і WeakSet WeakMap і WeakSet - «урізані» по функціоналу варіанти Map / Set, які дозволяють тільки «точково» звертатися до елементів (по конкретному ключу або значенням). Іншими словами, WeakMap/WeakSet працює тільки на запис (set, delete) і читання (get, has) елементів по конкретному ключу, можна додавати елементи, перевіряти їх наявність, але не можна отримати їх список і навіть дізнатися кількість.
Arrow Functions Arrow Functions don't have their own context this and arguments, they take this, arguments from outer context.
Classes ES6 classes are a simple sugar over the prototype-based OO pattern. Having a single convenient declarative form makes class patterns easier to use, and encourages interoperability. Classes support prototype-based inheritance, super calls, to instantiate parent properties.
Promises Promise gives us control over our async function and allows us to either resolve or reject an operation request.
Modules in ES6 There were no native modules support in JavaScript before ES6, in ES5 you would use tags with IIFE, or some library like AMD, while in ES6 you can use modules with IMPORT and EXPORT operands
String Interpolation Intuitive expression interpolation for single-line and multi-line strings.
Generators Генератори - новий вид функцій в сучасному JavaScript. Вони відрізняються від звичайних тим, що можуть призупиняти своє виконання, повертати проміжний результат і далі відновлювати його пізніше, в довільний момент часу.
Для оголошення генератора використовується нова синтаксична конструкція: function *
Основним методом генератора є next (). При виклику він відновлює виконання коду до найближчого ключового слова yield. Після досягнення yield виконання призупиняється, а значення - повертається в зовнішній код. Повторний виклик generator.next () відновить виконання і поверне результат наступного yield.І, нарешті, останній виклик завершить виконання функції і поверне результат return.
Функція завершена. Зовнішній код повинен побачити це з властивості done: true і обробити value: 3, як остаточний результат. Нові виклики generator.next () більше не мають сенсу. Втім, якщо вони і будуть, то не викличуть помилки, але будуть повертати один і той же об'єкт: {done: true}.
find Lets you iterate through an array and get the first element back that causes the given callback function to return true. Once an element has been found, the function immediately returns. It’s an efficient way to get at just the first item that matches a given condition.
findIndex FindIndex behaves similar to find, but instead of returning the element that matched, it returns the index of that element.
Array.from Array.from() enables the creation of a new array from an array like object.
Array.of Array.of () create a new instance of Array with an arbitrary number of arguments, regardless of the number or type of argument.
fill Fills an array with the given value

ES7 - ES8 features

Feature Descrption Code
includes Checks the array for the value passed as an argument. It returns true if the array contains the value, otherwise, it returns false.
String padding This section adds two functions to the String object: padStart & padEnd. As their names, the purpose of those functions is to pad the start or the end of the string, so that the resulting string reaches the given length.
Object.values The Object.values method returns an array of a given object’s own enumerable property values, in the same order as that provided by a for in loop.
Object.entries The Object.entries method returns an array of a given object's own enumerable property [key, value] pairs, in the same order as Object.values.
Object.getOwnPropertyDescriptors The getOwnPropertyDescriptors method returns all of the own properties descriptors of the specified object. An own property descriptor is one that is defined directly on the object and is not inherited from the object’s prototype.
Trailing commas in function parameter lists and calls Trailing commas in function parameter is the ability of the compiler not to raise an error (SyntaxError) when we add an unnecessary comma in the end of the list
Promise.Prototype.finally finally() method, which is a callback function for when the Promise is fulfilled or rejected. We can expect the same behavior of the try/catch/finally, where finally will always be executed
async/await - async is a keyword for the function declaration
- await is used during the promise handling
- await must be used within an async function, though Chrome now supports "top level" await
- async functions return a promise, regardless of what the return value is within the function
- async/await and promises are essentially the same under the hood
